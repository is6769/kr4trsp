# FastAPI Контрольная работа №4

Комплексное решение для заданий 9.1, 10.1, 10.2, 11.1, 11.2.

## Установка

```bash
pip install -r requirements.txt
```

## Конфигурация

Скопируйте `.env.example` в `.env`:

```bash
cp .env.example .env
```

## Запуск приложения

```bash
uvicorn app.main:app --reload
```

Приложение будет доступно на `http://localhost:8000`

## Миграции БД (Задание 9.1)

Инициализация Alembic (уже выполнено):

```bash
alembic upgrade head 
```

Создание новой миграции:

```bash
alembic revision --autogenerate -m "Описание миграции"
```

Откат миграции:

```bash
alembic downgrade -1
```

## Тестирование

### Синхронные тесты (Задание 11.1)

```bash
pytest tests/test_sync.py -v
```

### Асинхронные тесты (Задание 11.2)

```bash
pytest tests/test_async.py -v
```

### Все тесты


## Реализованные задания

### ✅ Задание 9.1 - Alembic миграции
- Настройка Alembic в проекте FastAPI
- Модель Product с полями: id, title, price, count
- Первая миграция (создание таблицы)
- Вторая миграция (добавление поля description)

### ✅ Задание 10.1 - Кастомная обработка ошибок
- CustomExceptionA (код 400) - общая ошибка
- CustomExceptionB (код 404) - ресурс не найден
- Обработчики исключений для каждого типа
- Эндпоинты, демонстрирующие использование

### ✅ Задание 10.2 - Валидация данных
- Модель User с Email валидацией
- Проверка age > 18, password 8-16 символов
- Кастомный обработчик ошибок валидации
- Эндпоинт для создания пользователя

### ✅ Задание 11.1 - Синхронные тесты
- Тесты для create_user, get_user, delete_user
- Тестирование успешных сценариев
- Тестирование ошибочных сценариев (404, 422)
- Проверка кодов состояния и структуры ответа

### ✅ Задание 11.2 - Асинхронные тесты
- Использование pytest-asyncio и httpx.AsyncClient
- ASGITransport для обращения к приложению напрямую
- Faker для генерации тестовых данных
- Изоляция состояния между тестами
- Покрытие успешных и ошибочных сценариев
